Help Guide

How to use ALEKS Calendar

Start Here

Use this order when setting up the app:

  1. Create a cycle
  2. Create a class
  3. Add students to that class
  4. Set cycle goals
  5. Begin entering daily reports

A good starting cycle is about two weeks. After enough data has been collected, you can adjust goals more accurately.

Cycles

A cycle is the date range you want to track.

Examples:

  • March 17 to March 28
  • April 1 to April 12

Goals, daily reports, pace, and projected progress are all measured inside the selected cycle.

Classes

Create each class you teach under the Manage tab.

Examples:

  • 61A
  • 61B
  • 62A

Students are then assigned to one class.

Students

Add each student with:

  • their name
  • their class

A student should belong to one class only.

You can edit student names and move them to a different class later if needed.

Daily Reports

For each lesson day, record:

  • Date
  • Minutes worked in ALEKS
  • Topics completed
  • Note if useful
  • Present or Absent

If a student attended but did not work, they can still be marked Present with 0 minutes or 0 topics.

If the student was not in class, mark them Absent.

Knowledge Checks

Knowledge checks are different from normal topic completion.

A student may complete very few topics during a lesson but still master new material through a knowledge check.

Use the Knowledge check option when that happens, then enter:

  • New topics mastered

Example:

  • Topics completed = 0
  • Knowledge check = yes
  • New topics mastered = 12

Focus / Disruption

These are optional.

Focus scale:

  • 1 = very low focus
  • 5 = very strong focus

Disruption scale:

  • 0 = no disruption
  • 5 = severe disruption

If you do not want to record these on a day, leave them off.

Dashboard

Select:

  • Cycle
  • Class
  • Student

The dashboard then shows:

  • Goal = target for the cycle
  • Completed = topics done so far
  • Remaining = still needed to reach goal
  • Pace = average topics per present logged day
  • Projected = estimated cycle finish based on current pace
  • Absences = days marked absent
  • New topics mastered = from knowledge checks

Progress Overview

This tab gives a class-level visual comparison.

Select a cycle and class to compare students by:

  • Goal
  • Current progress
  • Projected progress

Use this to quickly see who is on track, behind, or likely to exceed goal.

Tips

  • Create the cycle first before adding reports.
  • Use absences only when the student was truly not present.
  • Use notes for useful context, not for repeating the numbers.
  • Keep topics completed separate from knowledge-check gains.
  • Review goals after enough real data has been collected.
