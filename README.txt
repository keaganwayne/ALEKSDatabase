# ALEKS Calendar Guide
### Teacher setup and daily use

This tool helps you track ALEKS progress by **cycle**, **class**, and **student**, while recording daily lesson information in a way that is simple to update over time.

---

## What this tool is for

The ALEKS Calendar is designed to help teachers:

- organize students by class
- create a teaching cycle for a period of weeks
- assign topic goals for that cycle
- record daily ALEKS progress
- track absences, focus, disruption, and knowledge checks
- view projected progress toward goals
- compare students visually in the Progress Overview tab

---

## The basic flow

The app is built around this order:

**Cycle → Class → Student → Goals → Daily Reports**

That means the best setup order is:

1. Create a **cycle**
2. Create a **class**
3. Add **students** into that class
4. Set **cycle goals**
5. Begin recording **daily reports**

---

# First-time setup

## Step 1: Create a cycle
Go to the **Manage** tab.

Under **Cycles**, create the date range you want to track.

A cycle is the period of time you want to monitor before reviewing progress and adjusting goals.  
A good example is a **2-week cycle**.

Example:
- Start date: March 17
- End date: March 28

This gives you a clear block of time to observe student output before making major changes.

### Why start with a cycle?
Because all progress, goals, and projections are measured inside a cycle.

---

## Step 2: Create your class
Still in the **Manage** tab, go to **Classes**.

Add each class you teach, for example:

- 61A
- 61B
- 62A

Each student belongs to **one class only**.

---

## Step 3: Add students
Under **Students**, enter:

- the student’s name
- the class they belong to

Do this for all students you want to track.

### Tip
Check spelling carefully before saving.  
Student names can be edited later, but getting them right early keeps records cleaner.

---

## Step 4: Set cycle goals
Still in **Manage**, go to **Cycle goals**.

Choose:
- the cycle
- the class

Then either:
- set goals one student at a time, or
- use **Apply goal to all shown students** if the whole class starts with the same target

### Best practice
At the start of a cycle, set a reasonable goal based on your expectations.  
After about two weeks of tracking, use the actual data to adjust goals more accurately.

---

# Daily use

## Step 1: Go to Dashboard
On the **Dashboard** tab, select:

1. Cycle
2. Class
3. Student

The dashboard will then show that student’s data for the selected cycle.

---

## Step 2: Record a daily report
Under **Add / edit daily report**, enter the day’s information.

You can record:

- **Date**
- **Minutes**
- **Topics**
- **Note**
- **Attendance**
- optional **Focus**
- optional **Disruption**
- optional **Knowledge Check**

---

# What each field means

## Minutes
How many minutes the student was actively working in ALEKS during class.

This can be:
- a full lesson amount
- a partial amount
- **0 minutes** if the student attended but did not work

## Topics
How many ALEKS topics the student completed that day.

## Note
Use this for context that numbers alone do not show.

Examples:
- distracted by friends
- worked well after moving seats
- struggled with fractions
- did not bring computer
- needed frequent reminders

## Attendance
There are two options:

### Present
Use this when the student attended class.

### Absent
Use this when the student was not at school or not in class.

Absent records are **not counted** in performance calculations.

That means absences do not unfairly lower pace or projected progress.

---

# Focus and disruption

These are optional.

## Track focus
Use this when you want to rate how well the student focused during the lesson.

Scale:
- **1** = very low focus
- **5** = very strong focus

## Track disruption
Use this when you want to record how disruptive the student was.

Scale:
- **0** = no disruption
- **5** = very disruptive

### Important
These are optional for each entry.  
If you do not want to track them on a given day, leave them off.

---

# Knowledge checks

A knowledge check is different from normal topic completion.

Sometimes a student completes few or no topics during class because ALEKS gives a knowledge check instead.  
However, that knowledge check may still result in **new topics mastered**.

## When to use Knowledge Check
Tick **Knowledge check** when the student is doing a knowledge check that day.

Then enter:
- **New topics mastered**

### Example
A student may:
- complete 0 topics
- but master 12 new topics from the knowledge check

That should be recorded as:
- Topics = 0
- Knowledge Check = yes
- New topics mastered = 12

### Why this matters
It keeps daily topic completion separate from ALEKS reassessment growth.

---

# Understanding the Dashboard

The student dashboard shows:

## Goal
The total target number of topics for the cycle

## Completed
The number of topics completed so far in the cycle

## Remaining
How many topics are still needed to reach the goal

## Pace
Average topics completed per present logged day

## Projected
Estimated number of topics the student is likely to complete by the end of the cycle, based on current pace

## Absences
How many days were marked absent

## New topics mastered
Total topics gained through knowledge checks

---

# Daily report strip

The colored date strip shows lesson days in the cycle.

- **Green** = future lesson day
- **Blue** = today
- **Red** = past lesson day

A checkmark means that a report has already been recorded for that date.

This helps you quickly see:
- which days are complete
- which days still need entries

---

# Existing entries

The **Existing entries** section lets you review previous records for the selected student.

You can:
- edit an entry
- delete an entry

### Important
If you delete an entry, the app should keep you on the same selected student and cycle so you do not lose your place.

---

# Progress Overview tab

The **Progress Overview** tab gives a class-level visual summary.

Select:
- a cycle
- a class

You will then see a chart comparing students using:
- **Goal**
- **Current**
- **Projected**

This is useful for quickly spotting:
- students falling behind
- students likely to meet goal
- students exceeding expectation

---

# Recommended workflow for teachers

## At the start of a new cycle
1. Create the cycle
2. Create or confirm your classes
3. Add or confirm students
4. Set topic goals
5. Begin daily tracking

## During the cycle
- record one daily report per student per lesson day
- mark absences properly
- record knowledge checks when they happen
- use notes for meaningful behavior or learning context

## After about two weeks
Review:
- pace
- completed topics
- projected progress
- absences
- knowledge check gains

Then decide whether to:
- keep goals the same
- increase goals
- reduce goals
- adjust support for certain students

---

# Tips for accurate use

## Use “Absent” only for actual absence
Do not use absent for a student who attended but did no work.

If they were in class but did nothing:
- mark them **Present**
- use **0 minutes** or **0 topics** if appropriate
- explain in the note

## Use notes sparingly but clearly
A short useful note is better than a long vague one.

Good:
- Off task most of lesson, needed repeated reminders
- Strong focus today, completed work independently
- Knowledge check completed, 8 new topics mastered

Less useful:
- Bad day
- Fine
- Okay I guess

## Review goals based on evidence
Do not guess too early.  
Use actual cycle data before making strong changes to targets.

---

# Common mistakes to avoid

## Creating students before classes
Make the class first so students can be assigned correctly.

## Mixing up topics completed and new topics mastered
Keep them separate:
- **Topics completed** = normal ALEKS work
- **New topics mastered** = result from knowledge check

## Marking present students as absent
Absent days are excluded from calculations, so use that only when the student truly missed class.

## Ignoring absences when reviewing progress
A student with weak totals but many absences may need a different interpretation than a student who attended every day.

---

# Suggested routine

A simple routine that works well:

### Beginning of cycle
Set up cycle, class, students, goals

### Each lesson
Record daily reports

### End of each week
Check dashboard and projections

### End of cycle
Review actual progress and adjust next cycle goals

---

# Final note

This tool is meant to support teacher judgment, not replace it.

The numbers are useful, but they make the most sense when combined with:
- what you observed in class
- student effort
- attendance
- knowledge checks
- and the quality of work habits over time
